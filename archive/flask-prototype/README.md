# Flask プロトタイプ（参考用・凍結）

これは「Slay the Spinner」のゲームデザインを検証した最初の実装です。
物理演算はサーバーサイドで一括計算し、クライアントはCSS `@keyframes`で
その結果を再生するだけの仕組みで、リアルタイムなクライアントゲームでは
ありません。ブラウザ／Steam配信を目指す本実装は `godot/` 以下で
Godotベースに作り直されています。

**このディレクトリは参考資料としてのみ残しています。機能追加はしないでください。**
新しい実装で移植・再検証する際の仕様リファレンス、および挙動の
突き合わせ用（`simulation.py`をそのまま実行して基準トラジェクトリを
生成し、Godot版の物理演算と数値比較する等）に使います。

## ローカルでの起動方法

```bash
python3 -m venv .venv
.venv/bin/pip install flask numpy
.venv/bin/python app.py
# http://127.0.0.1:5000/ で確認
```

## 既知の問題

- `templates/simulation.html` の壁描画（SVGポリゴン）は、各壁の内側の頂点が
  中心方向へ1単位分しかオフセットされていない一方で、両端の頂点はアリーナ
  全幅（10単位）に広がる座標計算になっており、`scale=50`の下では50pxしか
  内側にない巨大な三角形になり、4枚の壁が重なり合って中央が塗りつぶされた
  ように見えるバグがある。Godot版では単純な`Line2D`矩形として作り直し、
  このロジックは踏襲しない。
